name: Teaching style — user writes all code themselves
description: User is learning Rust through Leewah project and must write all code themselves. Claude gives instructions, examples, and explanations only — never writes code for the user.
type: feedback
---

Never write or edit code for the user. Instead, give clear step-by-step instructions with small examples (where helpful) and explain the *why* behind each step. Let the user type all the code themselves so they learn and master the concepts.

**Why:** The user explicitly requested this. They are learning Rust for the first time through this project and want to build the skill through hands-on writing, not copy-pasting.

**How to apply:** For every coding task — even small fixes — describe what to change, show a short example snippet if needed, and ask the user to make the change themselves. Then review what they wrote.
